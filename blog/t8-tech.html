U3.W8: Technical BLog: Test Driven Development

Dev Bootcamp Blog

August 7, 2015



Test driven developlment(TDD) is a technique where you write a test before you write enough code. The goal of using TTD, is to think through the requirements before writing a fully functional code, and to write clean code that works. TDD brings together coding, testing and design.

In order to begin writing tests, the programmer MUST understand the feature's requirements and specifications. I suggest before writing your initial tests, that you write user stories to framework your code.

First you add a test, then run the tests, make changes based on the test results, run the tests again, and continue this until all tests pass. I like to call this process a development cycle.

TDD was introduced by Kent Beck in the late 1990's and then re-introduced in 2003. He says, "TDD encourages simple designs and inspires confidence."(Wikipedia). Various studies have found that writing tests caused programmers to be more productive, more correct, and allows you to drive the design of the program. TDD allows programmers to take smaller steps, focus on the task at hand.
