U3.W7: Technical Blog: Ruby vs. JavaScript

Dev Bootcamp Blog

August 2, 2015



JavaScript and Ruby have many similarities within their programs. This post will address one fundamental difference between how JavaScript and Ruby do something. Today we are going to talk about Ojects vs Hashes.

In Ruby, a hash is a collection of unique keys and their values. They are a type of Array that allows you to use any object type. Hashes enumerate their values in the order that they keys were inserted. An example of a hash would be: 

ages ={
      "evian" => 23
      "laura" => 23
      "wendy" => 66
      "pat" => 97
      "moose" => 10
    }

I created the hash using "ages", which suggests that the hash will inlcude information about ages. I set the name "evian", which is the key, to 23, which is the value.

In JavaScript, the most similar thing to a hash is an object. An object contains name:value pairs, which are called property. You can access a hash by calling it like: age["evian"] would produce 23.

var ages = {
    name: "Laura",
    height: 61,
    sport: "shotput",
    quote: "yay"
  }

This is an example of an object in Javascript. An object needs to be set to a variable, in this case, var ages = the object. You can access a hash by calling it like: age.evian would produce 23.

The biggest difference between these two is that a JavaScript object is a collection of properties that reference other objects, and a Ruby has contains a hash of instance variables that reference other objects. That is why you cannot access a hash and an object in the same way.
