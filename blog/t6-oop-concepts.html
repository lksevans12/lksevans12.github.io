U2.W6: Technical Blog: Class Methods

Dev Bootcamp Blog

July 25th, 2015



A class method is a method that are called on in a class. An easy way to identify a class method is by seeing self.method_name. Using self indicates that the method_name is being defined in the class itself, not its instances. Self is always pointing to the current ojbect. In the example below, test is the class, .class_method is the class method. By adding self to the .class_method, I am refering to the object that holds the class structure.

class Test
      def self.class_method
      "I'm a class method."
      end
      end

You can also replace self with the object, in this case Test, and the code will work as well. Some people prefer this way, however it comes down to preference. Class methods only exist in the object that defined them. Class methods are used for anything that does not deal with an individual instance of a class.

The other style of using class method is the class << self. Using this style seperates class and instance method by using the wrapper, which improves code readability. It is also easier to add new class methods -- and you don't have to remember to add self. to each method. Finally, using this style is easier to refactor. An example of this style can be seen below.

While there are cons and pros to both styles, either will be correct in your code.
